构建依赖的专题理解

太灰2022-05-12 08:00:00思考学习前端 技术栈

# 两种依赖

一个node package有两种依赖：

dependencies 程序正常运行需要的包。 devDependencies 是开发需要的包，比如 一些单元测试的包之类的。

# 理解应用

如果将包下载下来在包的根目录里运行 npm install 默认会安装两种依赖，如果只是单纯的使用这个包而不需要进行一些改动测试之类的，可以使用 npm install --production ，只安装dependencies而不安装devDependencies。

  • 如果是通过 npm install packagename 那么只会安装到 dependencies。
  • 如果只想要安装 devDependencies ，需要输入 npm install packagename --dev。

# 相关举例

以下类库都建议安装到devDependencies:

例如 一些babel编译之类的 babel-plugin-component，eslint ，prettier ，less等

  • 单元测试支撑（mocha、chai）；
  • 语法兼容（babel）；
  • 语法转换（jsx to js、coffeescript to js、typescript to js）
  • 程序构建与优化（webpack、gulp、grunt、uglifyJS）；
  • css 处理器（postCSS、SCSS、Stylus）；
  • 代码规范（eslint）；

而像UI框架，JS框架，工具等，需要装到dependencies，例如 axios，element-ui， vue-router
